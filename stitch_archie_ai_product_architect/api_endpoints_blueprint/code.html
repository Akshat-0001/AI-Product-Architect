POST

/auth/login

Authenticate a user and return a JWT bearer token for subsequent requests. Requires a valid email and password combination.

Request Headers

Content-Type

Application media type

application/json

Accept

Response media type

application/json

Body Schema (JSON)

email string Required

The registered email address for the account.

password string Required

Minimum 8 characters, must include numbers.

remember_me boolean

Extends session validity to 30 days.

Response Status

200 Success: User logged in
401 Unauthorized: Invalid credentials
422 Validation: Invalid email format
Request Example content_copy 
{
  "email": "arch.alpha@example.com",
  "password": "********",
  "remember_me": true
}
Response (200 OK) content_copy 
{
  "status": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR...",
    "expires_at": "2023-12-31T23:59:59Z",
    "user": {
      "id": 1024,
      "name": "Architect Alpha"
    }
  }
}
Architect Tip

Consider implementing Rate Limiting (50req/min) on this endpoint to prevent brute-force attacks on user credentials.
